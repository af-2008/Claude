## Purpose

Track content that underperformed or was rejected — to prevent repeating the same mistakes. The Memory Agent reads this file and flags specific elements to avoid when generating new content. Without this, the same weak formats and hook types get recycled.

**Write rules:**
- Entries are added by Workflow 09 (write mode) only — do not manually edit unless correcting an error
- Append only — never delete or overwrite existing entries
- Every entry must include a specific hypothesis (what likely caused the failure) — not just "it didn't perform well"
- Every entry must be dated
- Only log posts that clearly underperformed (not average — genuinely weak)

---

## Entry Format

```
## [Date] — [Platform] — [Content Type]
Concept: [brief description of what the post was about — 1 sentence]
Hook used: "[exact hook text — word for word as it appeared]"
Metrics: Reach [#] | Likes [#] | Saves [#] | Shares [#]
What likely failed: [specific hypothesis — hook type, topic irrelevance, format mismatch, posting time, wrong audience, production quality, etc.]
Do not repeat: [specific element to avoid — e.g., "generic benefit statement hooks", "technique posts without local angle", "promotional posts on Monday mornings"]
```
